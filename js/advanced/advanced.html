  1. 
var x = 5;
function first() {
    console.log(x); //5
    var y = 10;

    function second() {
        console.log(y); // 10
        console.log(z); // undefined
        var z = 20;
    }

    second();
}
first();
console.log(y); //ReferenceError: y is not defined
  2. When second() is invoked, JavaScript hoists the declaration of z but not its initialization,so its value is undefined at that place
  3. In JS, scope chaining is a concept to resolve access of variable. When a function tries to access a variable, JS first searches this into local scope, if not found then it looks for parent's scope. In case of console.log(y), into second(), there is no variable y, so it searches for parent's scope and finds that y = 10, so it will gives us console.log(y) = 10
  4. When we console variable "y" out of the function first(), it will gives error: "ReferenceError: y is not defined". In JS variable scope is limited to function only, not outside of the function, That is why when we trying to access y outside function, it throwing error.
    Inshort, since y is not defined in the global scope, instead it defined inside function first(), so will throws a ReferenceError, when we call outside of function first().
